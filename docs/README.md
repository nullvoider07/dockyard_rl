# Building the docs

The documentation is a [Sphinx](https://www.sphinx-doc.org/) site authored in
MyST Markdown. The API reference is generated by
[sphinx-autodoc2](https://sphinx-autodoc2.readthedocs.io/), which extracts
docstrings from the source **statically** — no import of `dockyard_rl` and none
of the runtime dependencies (torch, vLLM, jax) are required to build the docs.

```bash
pip install -r docs/requirements.txt
cd docs && make html
# open _build/html/index.html
```

`make clean` removes `_build/` and the generated `apidocs/`. Both are
git-ignored; the API pages are regenerated on every build from the source tree.

## Layout

| Path | Contents |
| --- | --- |
| `index.md` | Landing page and top-level table of contents. |
| `getting-started/` | Install, quickstart, configuration model. |
| `architecture/` | How the system fits together at runtime. |
| `design-docs/` | The "why" behind each subsystem. |
| `apidocs/` | Auto-generated API reference (build artifact). |

Authoring conventions: technical and concise, no decorative banners. This is a
standalone system — do not introduce references to the upstream projects it drew
lineage from.
